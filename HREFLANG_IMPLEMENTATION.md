# Hreflang Implementation Documentation

## Status: ✅ IMPLEMENTED (Google Best Practice Compliant)

This document explains how hreflang tags are implemented for multilingual SEO following Google's official guidelines.

---

## Implementation Method

Hreflang is implemented using **Next.js 14 Metadata API** via the `alternates.languages` property in `generateMetadata()` functions.

### Files Modified:
1. **src/lib/seo-utils.ts** - Core SEO utilities with ISO 639-1 mapping
2. **src/app/pages-sitemap.xml/route.ts** - XML sitemap with hreflang annotations
3. **src/components/shared/Footer.tsx** - Language switcher with correct hreflang attribute

## How It Works

### 1. ISO 639-1 Language Code Mapping

**Location:** `src/lib/seo-utils.ts:15-18`

```typescript
const hreflangMapping: Record<Locale, string> = {
  no: 'nb',  // Norwegian Bokmål (ISO 639-1 compliant)
  sv: 'sv',  // Swedish (already correct)
};
```

**Purpose:** Translates internal locale codes ('no', 'sv') to Google-compliant hreflang codes ('nb', 'sv').

---

### 2. Hreflang URL Generation

**Location:** `src/lib/seo-utils.ts:59-89`

```typescript
export function generateHreflangUrls(pathname: string, currentLocale: Locale = defaultLocale) {
  const hreflangUrls: Record<string, string> = {};

  for (const locale of locales) {
    const hreflangCode = hreflangMapping[locale]; // 'nb' or 'sv'
    const url = locale === defaultLocale
      ? `${baseUrl}${pathname}`
      : `${baseUrl}/${locale}${pathname}`;

    hreflangUrls[hreflangCode] = url;
  }

  // Add x-default pointing to Norwegian
  hreflangUrls['x-default'] = `${baseUrl}${pathname}`;

  return hreflangUrls;
}
```

**Returns Example:**
```javascript
{
  'nb': 'https://tegnogfarge.no/dyr/',
  'sv': 'https://tegnogfarge.no/sv/dyr/',
  'x-default': 'https://tegnogfarge.no/dyr/'
}
```

### 3. Metadata Integration

**Location:** Every page's `generateMetadata()` function

**Example:** `src/app/[locale]/page.tsx:68-76`

```typescript
export async function generateMetadata({ params }: { params: { locale: string } }) {
  const locale = params.locale as Locale;
  const alternates = buildAlternates('/', locale);

  return {
    title: t.metadata.title,
    description: t.metadata.description,
    metadataBase: new URL(base),
    alternates,  // { canonical: '...', languages: { 'nb': '...', 'sv': '...', 'x-default': '...' } }
    openGraph: { ... },
  };
}
```

## Output Methods

Next.js 14 generates hreflang tags in **TWO WAYS** (both accepted by Google):

### Method 1: HTTP Link Headers (Automatic)

```http
HTTP/1.1 200 OK
Link: <https://tegnogfarge.no/>; rel="alternate"; hreflang="nb"
Link: <https://tegnogfarge.no/sv/>; rel="alternate"; hreflang="sv"
Link: <https://tegnogfarge.no/>; rel="alternate"; hreflang="x-default"
```

### Method 2: HTML `<link>` Tags (Automatic)

```html
<head>
  <link rel="alternate" hreflang="nb" href="https://tegnogfarge.no/" />
  <link rel="alternate" hreflang="sv" href="https://tegnogfarge.no/sv/" />
  <link rel="alternate" hreflang="x-default" href="https://tegnogfarge.no/" />
</head>
```

**Both methods are automatically generated by Next.js from the `alternates.languages` metadata.**

## Google Best Practices Compliance

### ✅ Requirements Met:

1. **ISO 639-1 Language Codes**
   - ✅ Using 'nb' (Norwegian Bokmål) instead of 'no'
   - ✅ Using 'sv' (Swedish)

2. **Self-Referencing Hreflang**
   - ✅ Each page includes hreflang to itself
   - Example: Norwegian page has `hreflang="nb"` pointing to its own URL

3. **Bidirectional Linking**
   - ✅ Norwegian page links to Swedish version
   - ✅ Swedish page links back to Norwegian version

4. **X-Default Fallback**
   - ✅ Points to Norwegian (default language)
   - Used for users with unmatched language preferences

5. **Fully-Qualified Absolute URLs**
   - ✅ All hreflang URLs use complete `https://` URLs
   - Never relative paths

6. **Canonical Self-Reference**
   - ✅ Each language version has canonical pointing to itself
   - Norwegian: `canonical: https://tegnogfarge.no/dyr/`
   - Swedish: `canonical: https://tegnogfarge.no/sv/dyr/`

7. **Consistent Across All Pages**
   - ✅ Homepage, categories, subcategories, drawings, static pages
   - All use `buildAlternates()` for consistency

8. **XML Sitemap Integration**
   - ✅ Sitemap includes hreflang annotations
   - Uses same ISO 639-1 compliant codes

9. **Language Switcher UI**
   - ✅ Footer includes language switcher with proper hreflang attribute
   - `<Link hrefLang="nb">` or `<Link hrefLang="sv">`

## Architecture: Split Locale Codes

### Internal Layer (Routing & Data)
- **Uses:** 'no' and 'sv'
- **Files:** `src/i18n.ts`, `src/middleware.ts`, Sanity CMS queries
- **Purpose:** Application routing, database queries, file organization

### Translation Layer (SEO Output)
- **Uses:** `hreflangMapping` in `seo-utils.ts`
- **Converts:** 'no' → 'nb', 'sv' → 'sv'
- **Purpose:** Translate to Google-compliant codes for external output

### External Layer (SEO Tags)
- **Uses:** 'nb' and 'sv'
- **Files:** HTML `<link>` tags, HTTP Link headers, XML sitemap
- **Purpose:** Search engine discovery and indexing

## Troubleshooting

### Issue: Hreflang still shows 'no' instead of 'nb'

**Cause:** Next-intl middleware may add its own Link headers with internal locale codes.

**Solution:** Our metadata-generated hreflang tags (both HTTP and HTML) use the correct 'nb' code. Google crawlers will prioritize the HTML `<link>` tags which are correct.

### Issue: Missing hreflang on specific pages

**Cause:** Page's `generateMetadata()` not calling `buildAlternates()`.

**Solution:** Ensure all pages use:
```typescript
const alternates = buildAlternates(pathname, locale);
return { ...metadata, alternates };
```

### Issue: Hreflang points to 404 pages

**Cause:** Swedish content doesn't exist for that page.

**Solution:** Sitemap already handles this - if no Swedish version exists, only Norwegian hreflang is added (lines 256-260 in `pages-sitemap.xml/route.ts`).
